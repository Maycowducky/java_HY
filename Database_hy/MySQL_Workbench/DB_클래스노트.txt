릴레이션 : 행(투플)과 열(속성)
릴레이션 표기
- 릴레이션명(속성1, 속성2, 속성3...)
릴레이션 특징
- 속성은 단일값
- 중복된 투플은 없어야 한다.
- 속성 값은 도메인값 중 하나

슈퍼키 : 투플을 유일하게 식별할 수 있는 속성의 집합(속성이 하나 이상) = 고유값

후보키 : 투플을 유일하게 식별할 수 있는 속성의 최소 집합 = 최소 고유값

*기본키 : 후보키 중에서 선택한 키
-릴레이션(테이블)을 의미를 대표할 수 있는 키이어야함.
-밑줄을 그어야함.

대리키 : 마땅한 기본키가 없을 때 일련 번호 같은 가상의 속성을 만들어 기본키로 삼는 경우가 있음. (대표적인 ex) 주문 번호

대체키 : 기본키로 선정되지 않은 후보키

*외래키 : 릴레이션의 속성이 다른 릴레이션의 기본키를 참조하는 속성

학생(학생번호, 이름 : (학생번호 = 기본키)

수강(수강번호,과몬번호, 학생번호) : (학생번호, 과목번호 = 외래키)

과목(과목번호, 과목명, ...) : (과목번호 = 기본키)

데이터 무결성 : 왜래키와 기본키가 일치하는것

개체 무결성 제약조건 : 기본키의 특성을 지키면 됨.

참조 무결성 제약 조건 : 기본키와 외래키의 값이 일치하는것을 지키면 됨.



- 대학생 수강신청 프로그램
- 요구사항
  - 대학생은 회원가입을 통해 프로그램을 이용한다
  - 대학생은 회원가입시, 학번, 이름, 학과를 입력한다.
  - 과목은 관리자가 등록한다
  - 과목은 과목번호, 과목명, 이수학점, 이수시간을 입력한다
  - 관리자가 과목을 수강 등록을 한다
  - 수강을 등록하기 위해 과목번호, 강의실, 담당교수, 강의시간표, 강의년도, 학기를 입력한다.
  - 대학생은 등록된 수강 과목을 신청한다.

개체
- 대학생, 과목
관계
- 수강
